# 05 — 代码风格规范

## 1. 基本格式

| 规则 | 要求 |
|------|------|
| 缩进 | 4 个空格（不用 Tab） |
| 文件编码 | UTF-8 |
| 行宽 | 不超过 120 字符 |
| 大括号 | K&R 风格（左括号不换行） |
| 每行一条语句 | 不写 `a = b; c = d;` |
| 空行 | 方法之间空一行，逻辑块之间空一行 |

```java
// ✅ 正确
if (condition) {
    doSomething();
} else {
    doOther();
}

// ❌ 错误
if (condition)
{
    doSomething();
}
```

## 2. 命名规范

### 2.1 通用规则

| 元素 | 规则 | 示例 |
|------|------|------|
| 包名 | 全小写，点分隔 | `com.serenehealth.bean` |
| 类名 | 大驼峰 | `AppointmentActivity`, `DBHelper` |
| 方法名 | 小驼峰，动词开头 | `queryDoctorById()`, `cancelAppointment()` |
| 变量名 | 小驼峰 | `doctorList`, `userName` |
| 常量 | 全大写 `_` 分隔 | `DB_NAME`, `TABLE_APPOINTMENT` |
| 成员变量 | 小驼峰，不用 m 前缀 | `userId`, `doctorList` |

### 2.2 ViewBinding 使用规范

使用 ViewBinding 替代 findViewById 获取控件：

```java
// Activity 中声明 binding
private ActivityLoginBinding binding;

// Fragment 中声明 binding（onDestroyView 时置 null 防止内存泄漏）
private FragmentHomeBinding binding;
```

**控件 ID 命名**（XML 中 `android:id`）：

| 控件类型 | ID 命名 | 示例 |
|---------|--------|------|
| Button | `btn_含义` | `btn_submit`, `btn_login` |
| EditText | `et_含义` | `et_phone`, `et_password` |
| TextView | `tv_含义` | `tv_title`, `tv_doctor_name` |
| RecyclerView | `rv_含义` | `rv_doctor_list` |
| ImageView | `iv_含义` | `iv_avatar` |
| CheckBox | `cb_含义` | `cb_agree` |
| Spinner | `sp_含义` | `sp_department` |

**代码中访问控件**：

```java
// ✅ 通过 binding 对象访问
binding.btnSubmit.setOnClickListener(...)
binding.etPhone.getText().toString().trim()
binding.rvDoctorList.setAdapter(adapter)

// ❌ 禁止使用 findViewById（ViewBinding 已覆盖）
```

## 3. Activity 代码结构模板

```java
package com.serenehealth.activity;

import android.os.Bundle;
import android.widget.Toast;
import androidx.appcompat.app.AppCompatActivity;
import com.serenehealth.databinding.ActivityLoginBinding;
import com.serenehealth.db.DBHelper;
// ... 其他 import

public class LoginActivity extends AppCompatActivity {

    // ==================== 1. ViewBinding ====================
    private ActivityLoginBinding binding;

    // ==================== 2. 成员变量 ====================
    private DBHelper dbHelper;

    // ==================== 3. 生命周期 ====================
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        binding = ActivityLoginBinding.inflate(getLayoutInflater());
        setContentView(binding.getRoot());

        initData();
        setListeners();
    }

    // ==================== 4. 初始化数据 ====================
    private void initData() {
        dbHelper = DBHelper.getInstance(this);
    }

    // ==================== 5. 设置监听 ====================
    private void setListeners() {
        binding.btnLogin.setOnClickListener(v -> doLogin());
    }

    // ==================== 6. 业务方法 ====================
    private void doLogin() {
        String phone = binding.etPhone.getText().toString().trim();
        String password = binding.etPassword.getText().toString().trim();

        if (phone.isEmpty()) {
            Toast.makeText(this, "请输入手机号", Toast.LENGTH_SHORT).show();
            return;
        }
        if (password.isEmpty()) {
            Toast.makeText(this, "请输入密码", Toast.LENGTH_SHORT).show();
            return;
        }

        User user = dbHelper.getUserDao().login(phone, password);
        if (user != null) {
            Toast.makeText(this, "登录成功", Toast.LENGTH_SHORT).show();
            // 跳转主页...
        } else {
            Toast.makeText(this, "手机号或密码错误", Toast.LENGTH_SHORT).show();
        }
    }
}
```

### 代码顺序强制

```
1. ViewBinding 声明
2. 成员变量（DBHelper、Adapter 等）
3. 生命周期方法 (onCreate, onResume...)
4. initData()           — 初始化数据库、网络数据
5. setListeners()       — 设置点击监听
6. 业务方法（按页面操作流程排列）
```

> **注意**：ViewBinding 生成类命名规则为 `ActivityXxxBinding`（如 XML 名 `activity_login.xml` → 绑定类 `ActivityLoginBinding`）。无需单独写 `initViews()` 方法，XML 中按 ID 命名的控件直接通过 `binding.控件ID` 访问。

## 4. Fragment 代码结构模板

```java
public class HomeFragment extends Fragment {

    // 1. ViewBinding
    private FragmentHomeBinding binding;

    // 2. 成员变量
    private DBHelper dbHelper;
    private DepartmentAdapter adapter;

    // 3. onCreateView
    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container,
                             Bundle savedInstanceState) {
        binding = FragmentHomeBinding.inflate(inflater, container, false);
        initData();
        setListeners();
        return binding.getRoot();
    }

    // 4. initData
    private void initData() {
        dbHelper = DBHelper.getInstance(requireContext());
        adapter = new DepartmentAdapter();
        binding.rvDepartment.setLayoutManager(new LinearLayoutManager(getContext()));
        binding.rvDepartment.setAdapter(adapter);
    }

    // 5. setListeners
    private void setListeners() {
        binding.btnSearch.setOnClickListener(v -> searchDoctors());
    }

    // 6. 业务方法
    private void searchDoctors() { ... }

    // 7. 防止内存泄漏
    @Override
    public void onDestroyView() {
        super.onDestroyView();
        binding = null;
    }
}
```

## 5. 注释规范

### 什么时候写注释

```
✅ 写注释的场景：
  - 一段逻辑复杂、看代码不能立刻理解意图
  - 用到了非显而易见的业务规则
  - 临时解决方案 / TODO

❌ 不写注释的场景：
  - 方法名已经说清楚了（如 queryDoctorById）
  - getter / setter
  -  findViewById 这种标准写法
  - "声明按钮" "设置监听" 这种废话注释
```

### 注释格式

```java
// 单行注释：简短说明

// TODO 待完成的事项（全大写，方便全局搜索）
// TODO 2026-06-30 小明：号源不足时需弹出候补提示

/**
 * 类/方法的文档注释（仅在 DBHelper 等公共类使用）
 */
public class DBHelper extends SQLiteOpenHelper { ... }
```

## 6. MVC 分层约束

| 约束 | 说明 |
|------|------|
| Activity 不写 SQL | SQL 语句全部封装在 `db/` 包下的 XxxDao 中，Activity 只调方法 |
| XML 不写逻辑 | 布局文件纯 UI，不写 `android:onClick`，点击事件统一在 Java 中 `setOnClickListener` |
| Bean 不写方法 | 实体类只有 private 字段 + getter/setter，不写业务方法 |
| 不写死字符串 | 界面文字用 `@string` 资源引用，方便统一修改 |
| 不写死尺寸 | 间距、字号用 `dimens.xml` 管理 |
| 不写死颜色 | 颜色用 `colors.xml` 管理 |

## 7. 常见写法约定

### 7.1 Toast 提示
```java
// 统一用短提示
Toast.makeText(this, "提示内容", Toast.LENGTH_SHORT).show();
```

### 7.2 页面跳转
```java
// 显式 Intent，不传多余数据
Intent intent = new Intent(this, DoctorListActivity.class);
intent.putExtra("department_id", deptId);
startActivity(intent);
```

### 7.3 数据库调用
```java
// 通过单例拿到 DBHelper，再拿具体 DAO
DBHelper dbHelper = DBHelper.getInstance(this);
// Fragment 中用 requireContext() 或 getContext()
List<Doctor> doctors = dbHelper.getDoctorDao().queryDoctorsByDepartment(deptId);
```

### 7.4 列表适配器
```java
// RecyclerView 写法（配合 ViewBinding）
binding.rvDoctorList.setLayoutManager(new LinearLayoutManager(this));
DoctorAdapter adapter = new DoctorAdapter(doctorList, doctor -> {
    // 点击回调
});
binding.rvDoctorList.setAdapter(adapter);
```

## 8. 常见禁止项

| 禁止 | 原因 |
|------|------|
| `System.out.println()` | 用 `Log.d("TAG", msg)` 替代 |
| `e.printStackTrace()` | 用 `Log.e("TAG", "msg", e)` 替代 |
| 字符串用 `==` 比较 | 用 `"xxx".equals(str)` |
| Activity 中写 `new Thread()` | 需要异步时用 Handler + Thread 或 ExecutorService |
| 硬编码文件路径 | 用 `context.getFilesDir()` |
| `setOnClickListener` 嵌套超过 2 层 | 拆成独立方法 |
