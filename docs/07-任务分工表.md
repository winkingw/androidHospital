## 5. 公共模块（组长或队员A兼任）

| 任务 | 内容 | 优先级 |
|------|------|:--:|
| DBHelper 框架 | SQLiteOpenHelper 子类，建表 SQL，版本管理 | 🔴 最优先 |
| 全部 Bean 类 | 16 个实体类（14 必须 + 2 选做） | 🔴 最优先 |
| 全部 Dao 类 | 14 个 DAO（69 个方法） | 🔴 最优先 |
| MockDataUtil | 模拟数据初始化（文档06定义的全部数据） | 🔴 最优先 |
| SPUtil | SharedPreferences 工具类（登录态、首次启动标记等） | 🟡 次优先 |
| 资源文件 | colors.xml, strings.xml, dimens.xml（按 DESIGN.md 配置） | 🟡 次优先 |
| BaseActivity | 抽取公共方法（如获取 DBHelper 实例） | 🟢 可后补 |

> 公共模块**必须先于各队员开发完成**，否则其他人无法联调数据库。建议第一个冲刺周（Sprint 1）全组集中搞定公共模块。

## 6. 开发阶段规划

```
Sprint 1（第1周）：公共模块 + 基础框架
  ├── 全组：搭建项目骨架、Gradle配置
  ├── 组长：DBHelper + 全部 DAO + Bean + MockDataUtil
  └── 全组：熟悉代码框架、本地跑通

Sprint 2（第2周）：核心功能开发
  ├── 队员A：首页 1,2,3,4 (轮播图+科室+医生+预约)
  ├── 队员B：健康档案 11 + 后台 21,22 (科室医生维护)
  └── 队员C：个人中心 14,15,16 + 5 (登录+个人中心+预约记录+自助缴费)

Sprint 3（第3周）：扩展功能 + 联调
  ├── 队员A：首页 6,7,8,9 (消息中心+智能导诊+智能导航+健康讲堂)
  ├── 队员B：健康档案 12,13 + 后台 18,19,20,23,24,25
  └── 队员C：个人中心 17,27 + 10,26 (评价+医保卡+二维码+模拟支付)

Sprint 4（第4周）：测试 + 文档 + 答辩准备
  ├── 全组：交叉测试、修 bug
  ├── 全组：准备答辩材料（PPT、演示视频）
  └── 全组：写中期报告、实践报告
```
