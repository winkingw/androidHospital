# 01 — 技术栈选型与架构规范

## 1. 开发环境

| 项 | 版本 / 工具 |
|---|-----------|
| IDE | Android Studio Hedgehog (2023.1.1) 或更新版本 |
| JDK | JDK 11 |
| Gradle | 8.2+ |
| AGP (Android Gradle Plugin) | 8.2+ |
| 最低 SDK | API 24 (Android 7.0) |
| 目标 SDK | API 34 (Android 14) |
| 模拟器 | Pixel 系列 API 34 |

## 2. 技术栈

| 项 | 选型 | 说明 |
|---|------|------|
| 开发语言 | **Java** | 课程统一语言，与学长一致 |
| UI | **XML + ViewBinding** | 不用 DataBinding、不用 Jetpack Compose |
| 架构模式 | **MVC** | 传统 MVC：Activity 即 Controller |
| 数据库 | **SQLite + DBHelper** | 不用 Room，继承 SQLiteOpenHelper |
| 图片加载 | **Glide** | 加载轮播图、头像等 |
| 网络 | **不需要** | 所有数据本地模拟，不接后端 |
| 页面结构 | **Activity + Fragment** | 底部导航配合 Fragment 切换 |

## 3. MVC 架构约定

```
┌─────────────┐
│    View     │  XML 布局 + ViewBinding
│  (布局文件)  │  只负责展示，不写逻辑代码
└──────┬──────┘
       │ 用户操作
       ▼
┌─────────────┐
│ Controller  │  Activity / Fragment
│  (Activity) │  处理点击事件、调用 DBHelper、页面跳转
└──────┬──────┘
       │ 操作数据
       ▼
┌─────────────┐
│    Model    │  实体 Bean + DBHelper + DAO 操作
│  (数据层)    │  User、Doctor、Appointment 等实体类
└─────────────┘
```

### 分层职责

| 层 | 文件位置 | 负责 | 禁止 |
|----|---------|------|------|
| View | `res/layout/*.xml` | 界面布局、控件声明 | 不写 Java 逻辑 |
| Controller | `activity/` `fragment/` | 事件响应、调用数据库、跳转页面 | 不直接写 SQL，通过 DBHelper |
| Model | `bean/` `db/` | 实体类定义、建表 SQL、增删改查方法 | 不持有 Activity 引用 |

## 5. 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| Activity | `功能名Activity` | `LoginActivity`, `DoctorListActivity` |
| Fragment | `功能名Fragment` | `HomeFragment`, `ProfileFragment` |
| Bean | 名词，驼峰 | `User`, `Appointment`, `Department` |
| XML 布局 | `activity_功能.xml` | `activity_login.xml`, `activity_main.xml` |
| 控件 ID | `类型_含义` | `btn_submit`, `et_phone`, `rv_doctor_list` |
| DBHelper 方法 | `动词+名词` | `insertUser`, `queryAppointments`, `deleteSchedule` |
| Adapter | `数据类型Adapter` | `DoctorAdapter`, `AppointmentAdapter` |
| 常量 | 全大写 _ 分隔 | `DB_NAME`, `TABLE_USER` |
