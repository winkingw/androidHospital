### 核心关系速览

| 关系 | 说明 |
|------|------|
| 科室 1:N 医生 | 一个科室下有多位医生 |
| 医生 1:N 排班 | 一位医生有多个出诊排班记录（按日期+上下午） |
| 排班 1:N 号源 | 一个排班按20分钟拆分成多个预约号源 |
| 号源 1:N 预约 | 一个20分钟号源段可被多个用户预约（直到号源用完）|
| 用户 1:N 预约 | 一个用户可多次预约（用户即就诊人，不区分） |
| 预约 1:1 缴费单 | 一个预约对应一张缴费订单 |
| 预约 1:1 就诊历史 | 一个完成的预约产生一条就诊历史 |
| 用户 1:1 医保卡 | 一个用户绑定一张医保卡 |
| 医保卡 1:N 消费记录 | 一张医保卡有多笔消费 |
| 用户 1:N 消息 | 一个用户接收多条消息 |
| 用户 1:N 评价 | 一个用户可多次评价 |
| 管理员 1:N (无直接关联) | 管理员/医生通过 admin_user 登录后台 |

> **说明**：
> - 健康讲堂 → 跳转外部网站，不建表。
> - 健康档案"处方"和"报告" → 硬编码模拟数据展示，不走数据库。
> - 就诊人不单独建表，用户即就诊人（仅支持给自己挂号）。
> - t_help_content 为选做表，帮助/隐私/流程内容可硬编码在页面中。

## 2. 数据字典

### 2.1 t_user — 用户/就诊人表

> 用户即就诊人，个人信息直接存在用户表中。

| 字段名 | 类型 | 约束 | 字段说明 |
|--------|------|------|---------|
| id | BIGINT | PK，自增 | 用户ID |
| phone | VARCHAR(20) | 唯一，非空 | 手机号（登录账号） |
| password | VARCHAR(100) | 可空 | 密码；仅验证码登录时可先为空 |
| real_name | VARCHAR(30) | 非空 | 本人姓名（即就诊人姓名） |
| gender | TINYINT | 可空 | 0未知 / 1男 / 2女 |
| birth_date | DATE | 可空 | 出生日期，用于计算年龄 |
| id_card_no | VARCHAR(18) | 唯一，可空 | 身份证号，展示时脱敏 |
| health_score | INT | 可空 | 健康指数 |
| member_level | VARCHAR(20) | 默认'普通会员' | 会员等级 |
| create_time | DATETIME | 非空 | 创建时间 |
| update_time | DATETIME | 非空 | 更新时间 |

### 2.2 t_department — 科室表

| 字段名 | 类型 | 约束 | 字段说明 |
|--------|------|------|---------|
| id | BIGINT | PK，自增 | 科室ID |
| dept_name | VARCHAR(50) | 非空 | 科室名称 |
| dept_code | VARCHAR(50) | 唯一，可空 | 科室编码，便于后台管理 |
| parent_id | BIGINT | 可空，FK→t_department.id | 上级科室ID；无上级时为空 |
| description | VARCHAR(500) | 可空 | 科室简介 |
| sort_no | INT | 默认0 | 前端展示排序 |
| create_time | DATETIME | 非空 | 创建时间 |
| update_time | DATETIME | 非空 | 更新时间 |

### 2.3 t_doctor — 医生表

| 字段名 | 类型 | 约束 | 字段说明 |
|--------|------|------|---------|
| id | BIGINT | PK，自增 | 医生ID |
| department_id | BIGINT | FK→t_department.id | 所属科室ID |
| doctor_name | VARCHAR(30) | 非空 | 医生姓名 |
| gender | TINYINT | 可空 | 0未知 / 1男 / 2女 |
| title | VARCHAR(50) | 可空 | 职称：主任医师/副主任医师/主治医师 |
| introduction | TEXT | 可空 | 医生简介（含学历、擅长方向等） |
| status | TINYINT | 默认1 | 1在职可挂号 / 0停用 |
| sort_no | INT | 默认0 | 展示排序 |
| create_time | DATETIME | 非空 | 创建时间 |
| update_time | DATETIME | 非空 | 更新时间 |

### 2.4 t_doctor_schedule — 医生排班表

> 保存医生某日上午/下午是否出诊，为号源拆分提供时间范围。

| 字段名 | 类型 | 约束 | 字段说明 |
|--------|------|------|---------|
| id | BIGINT | PK，自增 | 排班ID |
| doctor_id | BIGINT | FK→t_doctor.id | 出诊医生ID |
| schedule_date | DATE | 非空 | 出诊日期 |
| period | VARCHAR(20) | 非空 | MORNING上午门诊 / AFTERNOON下午门诊 |
| clinic_room | VARCHAR(50) | 可空 | 诊室或楼层 |
| start_time | TIME | 非空 | 上午固定08:00:00，下午固定13:00:00 |
| end_time | TIME | 非空 | 上午固定12:00:00，下午固定17:00:00 |
| schedule_status | TINYINT | 默认1 | 1正常出诊 / 0停诊 |
| create_time | DATETIME | 非空 | 创建时间 |
| update_time | DATETIME | 非空 | 更新时间 |

> 唯一约束：(doctor_id, schedule_date, period)

### 2.5 t_register_source — 挂号号源表

> 排班下按 20 分钟拆分号源，每个时间段一条记录。

| 字段名 | 类型 | 约束 | 字段说明 |
|--------|------|------|---------|
| id | BIGINT | PK，自增 | 号源ID |
| schedule_id | BIGINT | FK→t_doctor_schedule.id | 所属排班ID |
| slot_start_time | TIME | 非空 | 时间段开始，如08:00:00、08:20:00 |
| slot_end_time | TIME | 非空 | 时间段结束，固定=start+20分钟 |
| total_num | INT | 非空，默认1 | 该时段总号源数（课程项目建议默认1） |
| remain_num | INT | 非空，默认1 | 剩余号源数；预约-1，取消+1 |
| register_fee | DECIMAL(10,2) | 默认20.00 | 该时段挂号费 |
| source_status | TINYINT | 默认1 | 1可预约 / 0不可预约 |
| version | INT | 默认0 | 乐观锁，防止抢号数据错误 |
| create_time | DATETIME | 非空 | 创建时间 |
| update_time | DATETIME | 非空 | 更新时间 |

> 唯一约束：(schedule_id, slot_start_time, slot_end_time)
>
> 上午时段拆分示例：08:00-08:20, 08:20-08:40, ..., 11:40-12:00（共12段）

## 3. 建表 SQL

```sql
-- ============================================
-- Serene Health 数据库建表脚本（最终版）
-- 数据库名: serene_health.db
-- ============================================

-- 1. 用户/就诊人表
CREATE TABLE t_user (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    phone         TEXT    NOT NULL UNIQUE,
    password      TEXT,
    real_name     TEXT    NOT NULL,
    gender        INTEGER,
    birth_date    TEXT,
    id_card_no    TEXT    UNIQUE,
    health_score  INTEGER,
    member_level  TEXT    DEFAULT '普通会员',
    create_time   TEXT    NOT NULL DEFAULT (datetime('now','localtime')),
    update_time   TEXT    NOT NULL DEFAULT (datetime('now','localtime'))
);

-- 2. 科室表
CREATE TABLE t_department (
```

## 4. 表汇总

| 序号 | 表名 | 中文名称 | 必须 | 对应功能 |
|------|------|---------|:--:|------|
| 1 | t_user | 用户/就诊人表 | ✅ | 登录注册、个人中心（用户即就诊人） |
| 2 | t_department | 科室表 | ✅ | 科室列表、后台维护科室 |
| 3 | t_doctor | 医生表 | ✅ | 医生列表、搜索、后台维护 |
| 4 | t_doctor_schedule | 医生排班表 | ✅ | 排班管理（上/下午出诊） |
| 5 | t_register_source | 挂号号源表 | ✅ | 20分钟号源段、预约挂号 |
| 6 | t_appointment | 预约挂号记录表 | ✅ | 预约提交、取消、预约记录 |
| 7 | t_payment_order | 缴费订单表 | ✅ | 自助缴费、模拟支付 |
| 8 | t_medical_card | 模拟医保卡表 | ✅ | 医保卡绑定、卡面展示 |
| 9 | t_medical_card_record | 医保消费明细表 | ✅ | 医保消费记录 |
| 10 | t_visit_history | 就诊历史记录表 | ✅ | 健康档案-就诊历史 |
| 11 | t_message | 消息通知表 | ✅ | 消息中心（含已读/未读） |
| 12 | t_banner | 首页轮播图表 | ✅ | 轮播图管理 |
| 13 | t_feedback | 满意度评价表 | ✅ | 三维评分（医生/服务/体验） |
| 14 | t_admin_user | 后台账号表 | ✅ | 管理员/医生端登录 |
| 15 | t_symptom_department_rule | 症状-科室匹配表 | 选做 | 智能导诊 |
| 16 | t_help_content | 帮助内容表 | 选做 | 帮助/隐私/流程（可硬编码替代） |

> **不建表的功能**：
> - 健康讲堂 → 跳转外部网站
> - 健康档案的处方/报告页签 → 硬编码模拟数据
> - 智能导航 → 跳转第三方地图（系统Intent）
